# /batch-loop

複数のactiveチケットを、各チケット完了時の承認ゲートで止めずに連続でループさせてください。docs/batch-loop.md（複数チケットの連続ループ実行ガイド）の命令テンプレートに相当するバッチ事前承認を組み立て、orchestratorとしてループを起動します。

## 手順

1. $ARGUMENTS からチケットID列を読み取る（スペース区切り・記載順が実行順）
   - 形式: `{チケットID} {チケットID} ...`（例: `APP-001 APP-002 APP-003`）
   - IDが1件以下の場合は連続実行の意味がないため、`/start-loop` の使用を案内して終了する
2. 各IDのチケットが tickets/active/ に存在し、statusが `cancelled` / `blocked` でないことを確認する。問題があれば該当チケットを一覧表示し、人間に対象の修正を求めて停止する
3. **バッチ事前承認の確認（必須）**: 以下を人間に提示し、承認を得てから開始する
   - 対象チケット（ID・タイトル・現status）と実行順
   - 各チケットが reviewer承認で done に到達したら、成果物の要約だけ報告し、改善ループ判断は「実装ループ継続」として扱い、停止せず次のチケットに進むこと
   - 最終チケットが done に到達したら、そこで停止して報告し、人間の判断を待つこと
   - この事前承認は今回列挙したチケット間の遷移のみに適用されること
   - 差し戻し（tester/reviewer）・リトライ上限超過・blocked が発生した場合は、この承認に関わらず直ちに停止して報告すること
4. .claude/agents/orchestrator.md を読み込み、.claude/commands/start-loop.md の起動時チェックリスト（Autoモード確認・14日blockedトリアージ・in_progress上限・done/件数）を実行する
5. 先頭チケットからループを開始し、手順3で承認された範囲で連続実行する

## 制約（docs/batch-loop.md 準拠）

- **逐次実行であり並行ではない** — `One ticket, one active sub-agent at a time`。1件を最後まで流してから次へ進む
- **報告は省略しない** — done到達ごとに成果物の要約を必ず報告する（`Skip reporting to human after reviewer approval` の禁止を守る。*待たない*だけにする）
- **事前承認は明示した遷移のみに有効** — 範囲外のチケットには通常どおり承認ゲートが入る
- **異常系では安全装置を効かせる** — 差し戻し・リトライ上限超過・blocked では直ちに停止して報告する

## 使用例

```
/batch-loop APP-001 APP-002
/batch-loop APP-001 APP-002 APP-003
```
