# /setup

Kenesis Loop Kitを導入したプロジェクトの初期セットアップと現在地診断を実行してください。

**全ステップは冪等です。** 各ステップは「状態を確認し、済みなら結果を報告してスキップ」で統一します。初回導入時だけでなく、途中まで進んだプロジェクトで再実行しても常に正しい「次の一手」を案内します。

## 手順

### 1. .gitignoreプリセットの選択

1. 現在の `.gitignore` を `.gitignore.public` / `.gitignore.private` と比較する
   - **どちらとも一致しない場合**（独自カスタマイズの可能性）は差分を提示し、上書きしてよいか人間に確認する。確認なしに上書きしない
2. 人間に「プライベートリポジトリで実チケット（tickets/active・done）もコミットしたいか」を確認する
   - はい → `cp .gitignore.private .gitignore`
   - いいえ / 不明 → 変更しない（デフォルトのpublicプリセットは実チケットを除外する安全側）

### 2. Obsidianの案内（任意）

- Obsidianは必須ではないことを伝える（実体はMarkdownファイル群であり、エディタだけで全機能が動作する）
- 使う場合の手順（`tickets/` をvaultとして開く・推奨プラグイン）は `docs/obsidian-setup.md` を案内する。GUI操作のためこのコマンドからは実行しない

### 3. プロジェクト略称の確定

1. CLAUDE.md「チケットID採番」節に「このプロジェクトの略称」の行があるか確認する
2. 既にあれば現在の略称を報告してスキップする
3. 無ければ人間にチケットIDの略称（例: `APP`）を確認し、同節へ以下の形式でEditツールで追記する（`/new-ticket` が採番と `project` フロントマターに使用する）

   ```
   - このプロジェクトの略称: `XXX`
   ```

### 4. SPEC・前工程の現在地診断

以下を上から順に判定し、**最初に該当したもの**を「次の一手」として案内する。このコマンドではSPEC.mdやチケットを作成せず、必ず専用のスキル/コマンドへ誘導する。

| 状態 | 次の一手 |
|---|---|
| `docs/SPEC.md` が無い | `/spec-interview` で要件定義を作成する |
| SPEC有り・`docs/spec-qa/<spec名>/` が無い | `/interrogate-spec docs/SPEC.md` でSPECを敵対的レビューする |
| 尋問済み・SPEC §6画面一覧に「(未作成)」がある、または `docs/wireframes/` に対応HTMLが無い | `/wireframe-gen` でワイヤーフレームを生成する |
| `tickets/active/` が0件（.gitkeepを除く） | `/plan-tickets` でSPECからチケットを一括起票し、`/start-loop`（連続実行は `/batch-loop`）でループを開始する。単発の起票は `/new-ticket` |
| activeチケットあり | `/start-loop` でループを再開する |

### 5. 結果報告

各ステップの実施/スキップ結果と「次の一手」を一覧にまとめて報告する。

## Never

- 人間の確認なしに `.gitignore` を上書きしない
- SPEC.md・ワイヤーフレーム・チケットをこのコマンドで直接作成・編集しない（手順4の誘導先に委ねる）
- CLAUDE.mdへの書き込みは手順3の略称行の追記のみ。他の節を変更しない

## 使用例

```
/setup
```
