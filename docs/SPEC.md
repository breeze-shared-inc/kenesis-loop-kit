## 1. プロジェクト概要

既存のWordPressコーポレートサイトのトップページをリニューアルし、第一印象（ファーストビュー）と
問い合わせ導線を改善する。あわせて表示速度とスマホ表示を改善し、訪問者が「何の会社か」を素早く理解し、
問い合わせ（CV）まで迷わず到達できる状態にする。

本サイトの事業は「ITインフラ」「SES（システムエンジニアリングサービス）」の二本立てであり、
トップページではこの2事業を明確に伝える。さらに新規の採用情報導線は、主にSES事業の人材確保を
目的として設計する。

---

## 2. 背景・目的

現状のトップページは表示が遅く、デザインも古いため第一印象が弱い。
ファーストビューで自社の価値が伝わりにくく、アクセスはあっても問い合わせにつながっていない。

本改善により、ファーストビューと問い合わせ導線を刷新して第一印象とCV導線を強化し、
表示速度・スマホ表示の改善で離脱を抑えることを目的とする。

あわせて、新規に採用情報（求人）への導線をトップページへ追加し、求人に関する問い合わせ・応募を
拡充する。サービスへの問い合わせ獲得だけでなく、人材確保（採用）も本改善の目的に含める。

採用の主目的はSES事業の人材確保にある。SESは稼働するエンジニアの人数が事業規模に直結するため、
ITエンジニア（特にインフラ系）の応募を継続的に獲得できる導線をトップページに設けることを重視する。
そのため、トップページでは「ITインフラ」「SES」の2事業をわかりやすく提示しつつ、
SES志望のエンジニアが採用情報へ迷わず到達できる導線を用意する。

## 3. ユーザー・ステークホルダー

| ユーザー種別 | 説明 | 主な操作 |
|---|---|---|
| 見込み顧客（サイト訪問者） | ITインフラ／SESの発注を検討する企業担当者 | トップページ閲覧・サービス問い合わせ |
| 求職者（SES志望のエンジニア） | SES事業への参画を検討するITエンジニア（特にインフラ系） | 事業内容・採用情報の閲覧・求人問い合わせ／応募 |
| 営業／問い合わせ担当 | サービス問い合わせを受け対応する社内担当 | サービス問い合わせ件数の確認（事業KPIオーナー） |
| 採用担当 | 求人問い合わせ・応募を受け対応する社内担当 | 応募件数の確認（採用KPIオーナー） |
| サイト運用担当 | WordPressでサイトを更新する担当 | コンテンツ更新・公開 |

## 4. 機能要件

### Must（必須）
- [ ] ファーストビューを刷新する（キャッチコピー・メインビジュアル・CTAの見直し）
- [ ] 問い合わせCTAの配置と文言を改善し、CVへの導線を強化する
- [ ] 主要スマホ幅でCTAが視認・タップしやすい状態にする
- [ ] トップページで「ITインフラ」「SES」の2事業を**対等の比重**でわかりやすく提示する
- [ ] トップページにSES事業向けの採用情報導線を新規追加し、**新規の採用ページ**（別チケットで構築）へ誘導する
- [ ] 求人問い合わせは**採用専用の受付窓口**へ誘導する（サービス問い合わせとは分離する）

### Should（できれば）
- [ ] 表示速度を改善する（画像最適化・不要スクリプト/プラグインの削減）
- [ ] スマホレスポンシブの崩れを解消する
- [ ] 採用情報導線のクリック（求人問い合わせへの遷移）を計測できるようにする

### Could（余裕があれば）
- [ ] 実績・お客様の声セクションを追加して信頼性を補強する
- [ ] SES志望者向けの訴求要素（働き方・案件例・キャリアパスなど）をトップページで簡潔に紹介する

## 5. デザイン・レイアウト要件

> 第一印象とCV／採用導線に直結するため、希望を具体的に記述する。
> 最終的な装飾・実装はarchitect/implementerが設計するが、本セクションを制約条件とする。

### 5.1 全体トーン・カラー
- 基調トーン: **信頼感・堅実さ**をベースとしつつ、**現代的・モダン**なデザインとする（BtoBのインフラ／SES企業にふさわしい落ち着き＋古さを感じさせない垢抜けた印象）。
- ただし**採用セクションのみ親しみやすさ**を出して使い分ける（人物写真・柔らかい余白・やや明るい配色）。
- **背景は白または白に近い色**を基調とし、余白を活かしたクリーンな見た目にする。
- メインカラー: **緑系を中心**とする（安心・成長の印象）。アクセントは1色に絞る。
- **過度な装飾はしない。** シンプルを基本とし、自然要素はポイント使いにとどめる（§5.5参照）。
- ロゴは**支給あり**。ロゴの色・余白規定があれば調査で確認し、緑系・白背景と整合させる。

### 5.2 ファーストビュー（FV）
- レイアウト: **左にテキスト**（大見出し・サブコピー・CTA）、**右にメインビジュアル**の2カラム。
- FV内に**2つのCTAを並置**する: 「お問い合わせ」（サービス）と「採用情報」（SES採用）。
- スマホでは1カラムに縦積みし、テキスト → ビジュアル → CTA の順とする。
- メインビジュアル（MV）には**自然・緑のモチーフ／画像**を用いる（§5.5参照）。
- メインビジュアルは表示速度目標と両立させる（画像最適化・遅延読み込みを前提とする）。

### 5.3 セクション構成（上から下への並び）
1. ファーストビュー（FV）— 3スライドのメインビジュアル（自動送り。一時停止/再生コントロールを備える）
2. 事業紹介 — 「ITインフラ」「SES」を**対等の比重**で並べる
3. 採用情報（SES）— 新規採用ページ／採用専用窓口への導線
4. 会社案内 — 理念・会社情報への導線
5. お知らせ — 最新のお知らせ（一覧ページへの導線）
6. お問い合わせ（サービス）
7. フッター

- 2事業は**同等の面積・視覚的重み**で提示し、優劣を感じさせないこと。
- 採用セクションは事業紹介の後に独立した塊として配置し、見落とされないようにすること。
- 「実績・信頼要素（旧Could）」は今回のトップ構成では採用しない（将来の追加候補として保留）。
- FVのメインビジュアルを自動送りカルーセルにする場合、恒常的な一時停止/再生手段を設ける（WCAG 2.2.2 Pause, Stop, Hide）。

### 5.4 その他のUI方針
- CTAボタンは色・サイズで明確に識別でき、スマホで親指が届く位置・大きさにする。
- 余白を十分にとり、情報を詰め込みすぎない（古い印象の払拭）。
- アニメーションは過度にせず、表示速度とアクセシビリティを優先する。
- フォント（書体）は指定なし。**architectが現サイト・ロゴとの整合を調査し、モダンで可読性の高い書体を提案**する（見出し・本文の組み合わせを含む）。
- 特定の参考サイトは指定なし。本セクション（§5）の記述をデザイン品質の基準とする。

### 5.5 ビジュアルモチーフ（自然・緑）
- メインビジュアル（MV）など大きなデザイン要素には、**緑を基調とした自然のモチーフ／画像**を用いる。
  - 例: 樹木・緑、青空、発芽する芽 など（成長・安心・信頼を想起させる要素）。
- **使い方はシンプル・ポイント使いを理想**とする。画面全体を装飾で埋めず、要所に自然要素を効かせる。
- 白〜白に近い背景と組み合わせ、自然要素が映える余白設計にする。
- **表現は「写真＋図形アクセント」を基本**とする。実写の自然・緑の写真をベースに、芽・曲線などの図形をポイントで重ね、モダンさと自然感を両立させる。図形アクセントは過剰にしないこと。

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | モバイルのLighthouse Performanceスコア90以上を目標とする（最低でも改善前から向上） |
| レスポンシブ | 360px〜の主要スマホ幅でレイアウト崩れがないこと |
| 対応ブラウザ | Chrome・Firefox・Safari・Edge の最新版（モバイルSafari/Chromeを含む） |
| アクセシビリティ | 主要テキストのコントラスト比がWCAG 2.1 AA相当を満たすこと |
| セキュリティ | WordPress本体・テーマ・プラグインを最新に保ち、機密情報をコードへハードコードしないこと |

## 7. 技術的制約

- CMS: WordPress（既存サイトを前提とする）
- 改修方針: 親テーマは直接編集せず、**子テーマ**で改修する（更新時の上書きを防ぐ）
- 言語: PHP / HTML / CSS / JavaScript
- 計測: アクセス解析（GA4を想定）にCTAクリックのイベント計測を追加する
- デプロイ: ステージング環境で確認してから本番へ反映する
- 既存の問い合わせフォーム（送信処理）の仕様は維持し、見た目と導線のみ改善する

---

## 8. スコープ外（明示的な除外事項）

- トップページ以外の下層ページ（会社概要・サービス詳細など）の改修
- 採用情報ページ本体（求人一覧・募集要項・応募フォーム）の新規構築。今回はトップページ上の導線追加までを対象とし、リンク先ページの整備は別チケットで扱う
- 多言語対応（日本語のみ）
- WordPress本体・サーバ・ドメインの移行
- 問い合わせフォームのバックエンド（送信・通知）ロジックの改修
- 新規機能の追加（ブログ機能・会員機能など）

---

## 9. 成功基準

- ファーストビューと問い合わせ導線がリニューアルされ、コードレビューが承認されていること
- 主要ブラウザおよびスマホ幅でレイアウト崩れがないことを目視検証済みであること
- 問い合わせCTAのクリックが計測できる状態になっていること（計測タグ設置済み）
- トップページに採用情報への導線が追加され、そのクリックが計測できる状態になっていること
- モバイルのLighthouse Performanceスコアが90以上であること（最低でも改善前から向上していること）

## 10. 未決事項（Open Questions）

- [ ] キャッチコピーの文言は誰が用意するか（社内 or 外注）未決定。※ロゴは支給済み
- [ ] MV用の自然・緑モチーフ素材の入手方法（支給／ストック購入／新規撮影・制作）未決定
- [ ] CTAクリック計測の解析ツールはGA4で確定か、別ツールを併用するか未決定
- [ ] 子テーマが未導入の場合、子テーマの新規導入を許容するか未決定
- [ ] 採用専用の受付窓口の実現方法（専用フォーム新設 or 採用ページに設置）は採用ページ構築チケットで詰める
- [ ] ファーストビューに重いメインビジュアルを使う場合、表示速度目標とのトレードオフをどう扱うか

### 決定事項（このSPECで確定したもの）

- トップページでは「ITインフラ」「SES」の2事業を **対等の比重** で提示する
- 今回の採用導線は **SES事業のみ** を対象とする（ITインフラ事業の採用は将来検討・スコープ外）
- 採用導線のリンク先は **新規の採用ページ**（自社サイト内）とする。ページ本体の構築は別チケット
- 求人問い合わせは **採用専用の受付窓口** に分離する（サービス問い合わせと混在させない）
- デザイン基調は信頼・堅実（緑系中心）、採用セクションのみ親しみやすさを出す（詳細は §5）
- FVは「左テキスト＋右ビジュアル」、FV内にサービス／採用の2CTAを並置する（詳細は §5.2）
