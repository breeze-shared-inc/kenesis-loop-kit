## 1. 目的

`docs/SPEC.md` §1・§4・§5 に基づき、コーポレートサイトのトップページのファーストビュー（FV）を刷新し、
「ITインフラ」「SES」の2事業を**対等の比重**（同等の面積・視覚的重み）で提示する。FV内に「お問い合わせ（サービス）」
「採用情報（SES採用）」の2CTAを並置し、訪問者・SES志望エンジニアの双方が迷わず目的の導線へ到達できる状態にする。
本チケットのスコープは **FVと2事業提示を中核**としつつ、トップページとしての体裁を整えるため
**会社案内（company）・お知らせ（news）の導線セクション**までを含む（採用ページ本体・採用専用窓口・GA4本実装・
お知らせ一覧／個別記事・問い合わせフォーム本体は別チケット）。

**前提（人間判断で確定。本設計はこれに従う）:**
- 実装形式は **静的HTML/CSS/JS**。SPEC §7 の「WordPress子テーマ改修」は本番運用の前提として保持しつつ、本学習サンプルは WP非依存の静的プロトタイプとして構築する（2026-06-25 確定）。
- 配置先は **`site/`** ディレクトリ（テンプレートの `src/` 前提を本設計で上書き）。
- アセットは **プレースホルダで進行**（緑系ダミー画像 or CSSグラデーション＋インラインSVG図形）。実素材が来たら差し替え可能な構成にする。
- 非機能要件（モバイルLighthouse 90目標／主要テキストWCAG AAコントラスト／360px崩れ無し）を **tester の検証対象**とする。
- **構成の正は採用ワイヤー＋news拡張**（2026-06-26 改善ループで確定。D1）。FVカルーセル（D2）・scroll-reveal（D3）を設計スコープに含める。

## 3. 設計方針

**採用する方針:**
- `site/` 配下に **素のHTML/CSS/JS** でトップページ1枚（`index.html`）を構築する。ビルドツール・フレームワーク・パッケージマネージャは導入しない（ダブルクリック or ローカル静的サーバで開け、Lighthouse/ブラウザで直接検証できることを最優先）。
- カラーは **CSSカスタムプロパティ（カラートークン）** に集約し、緑系メインカラー＋単一アクセントを `:root` で一元管理する。実素材・ブランドカラー確定時にトークン値の差し替えだけで反映できるようにする。
- レイアウトは **CSS Grid / Flexbox** を用い、メディアクエリで「2カラム（PC）→ 縦積み（スマホ）」を切り替える。モバイルファースト（既定を縦積みとし、`min-width` で2カラムへ拡張）で記述する。
- ビジュアルは **「プレースホルダ写真領域 ＋ インラインSVG図形アクセント」** の二層構成。写真は緑系プレースホルダ（CSSグラデーション or ダミー画像）で代替し、芽・曲線などの図形はインラインSVGで重ねる。実写真が来たら写真領域のみ差し替える。
- 装飾は最小限（SPEC §5.4「過度な装飾はしない」）。アニメーションは **`prefers-reduced-motion` を尊重し、軽量なものに限る**。本改訂で **FVカルーセル（D2）と scroll-reveal（D3）をこの「軽量アニメ」の許容範囲として正式にスコープ内**とする。ただし WCAG 2.2.2（Pause, Stop, Hide）等の運動・自動更新に関する達成基準を満たすことを必須条件とする（§9）。
- JSは最小限（モバイルメニュー開閉・CTAクリック計測フック・FVカルーセル・scroll-reveal）。重いライブラリは持ち込まない（依存0・defer。Lighthouse対策）。

**代替案と却下理由:**
| 案 | 却下理由 |
|---|---|
| WordPress子テーマ構成を模写 | PHP/WPランタイム無しでは描画・Lighthouse検証ができず、学習サンプルの検証性が落ちる（investigator R1/R3）。人間判断で静的方式に確定済み（U1）。 |
| React/Vue等のSPAフレームワーク | ビルド環境・依存が増え、Lighthouseのバンドルコストも上がる。1枚の静的トップに対し過剰。SPEC §6のパフォーマンス目標に逆行。 |
| CSSフレームワーク（Bootstrap等）導入 | 不要CSSの読み込みでLighthouseに不利。緑系トーン・余白設計を素のCSSで制御する方が要件（§5）に忠実。 |
| 配置先を `src/` のまま使用 | `src/` はLoop Kitフレームワーク用ツリー。人間判断で配置先 `site/` に確定済み（U3）。本設計でテンプレートの `src/` 前提を上書きする。 |
| FVカルーセル／scroll-reveal を不採用に戻す | 2026-06-25〜26 の人間進化を正とする方針（改善ループ D2/D3）。アニメは無効化ではなく a11y 要件を付けて存続させる。 |
| results（実績・信頼）セクションを維持 | 人間が採用ワイヤーで廃止し、company/news を採用（D1）。SPEC §5.3 も併せて改訂する（D4）。 |

## 4. 実装詳細

> **配置先は `site/`**（テンプレートの `src/` 前提を本設計で上書き。implementerは必ず `site/` 配下を対象とすること）。

**ファイル構成:**
```
site/
├── index.html              # トップページ本体（唯一のHTML）
├── css/
│   └── style.css           # カラートークン・レイアウト・レスポンシブ・カルーセル・reveal
├── js/
│   └── main.js             # メニュー開閉 + CTA計測フック + FVカルーセル + scroll-reveal
└── assets/                 # プレースホルダ（実素材が来たら差し替え）
    ├── logo-placeholder.svg
    └── favicon.svg
```
- `site/wireframe.html` は成果物から**除去**する（D5。設計検討用の素材であり本番DOMではない。必要なら `docs/` 配下へ移設してよい）。
- 実写真・実ロゴが支給され次第、`assets/` の差し替え＋必要なら `<img src>` / CSS変数の調整のみで反映できること。

**ヘッダーナビ項目（実物に整合）:**
- グローバルナビ = `お知らせ(#news) / 事業内容(#business) / 採用情報(#recruit) / 会社案内(#company) / お問い合わせ(#contact, CTA)`。
- スマホはハンバーガー（`aria-expanded` トグル・`is-open`・Esc/リンククリックで閉じる）。`min-width:768px` で横並び。

**HTMLセクション構造（`<body>` 内、上→下の並び。本改訂の正）:**
```
<header>            ロゴ（プレースホルダ）＋ナビ（スマホはハンバーガー）
<main>
  <section id="fv">          FV: 2カラム（左テキスト / 右ビジュアル）。2CTA並置。3スライドのカルーセル
  <section id="business">    事業紹介: ITインフラ / SES を対等の2カラム（同一構造のカード×2）
  <section id="recruit">     採用情報（SES）: 採用ページ/専用窓口への導線プレースホルダ。親しみやすいトーン
  <section id="company">     会社案内: 理念・会社情報への導線（テキスト＋ビジュアル2カラム・プレースホルダ）
  <section id="news">        お知らせ: 最新3件＋一覧リンク（ダミーデータ・プレースホルダ）
  <section id="contact">     お問い合わせ（サービス）: CTA。フォーム本体は別途（リンク/プレースホルダ）
</main>
<footer>            会社名・コピーライト・サイト内リンク群
```
- **`results（実績・信頼）` セクションは本改訂で廃止**（旧§9・SPEC §5.3 から削除）。代わりに `company` / `news` を配置する。
- 見出し階層は `h1`（FVの大見出し、ページ内1つ）→ 各 `section` 見出しを `h2` → カード見出しを `h3` とし、論理順序を保つ。各 `section` に `aria-labelledby` で見出しと関連付ける。

**FV／カルーセルのマークアップ・レスポンシブ方針（SPEC §5.2 / D2）:**
- DOM順は「テキスト → ビジュアル → CTA」（CSS無効時も要件順を維持）。PC（`min-width:768px`）は `grid-template-areas`（`"text visual" / "cta visual"`）で左テキスト・右ビジュアルの2カラム化。
- **カルーセル仕様（3スライド）:**
  - スライド構成: 1=会社全体 / 2=ITインフラ訴求 / 3=SES訴求。ビジュアル（`.fv__photo` 3レイヤー）はクロスフェード、テキストはフェードスルーで内容差し替え。
  - 自動送り間隔は **5.5秒**（`INTERVAL=5500`）。手動操作（ドット）後はタイマーをリセット。
  - `prefers-reduced-motion: reduce` 時は**自動送りせず即時切替**（既存挙動を維持）。
  - 外部依存0・`defer`・画像バイナリ0を維持（Lighthouse対策）。
- **a11y 必須修正（D2(a)(b)(c)。受け入れ条件化。§9参照）:**
  - **(a) 恒常的な一時停止/再生コントロール（WCAG 2.2.2 Pause, Stop, Hide）**: hover/focus による一時停止だけでは不十分。**常設の「一時停止／再生」ボタン**を FV内に追加する。要件:
    - キーボード到達可能・タップ領域 `≥44×44px`・`:focus-visible` 可視。
    - 状態を `aria-pressed`（押下=停止中）で表現し、ラベルを状態に応じて「一時停止する／再生する」に切替。
    - 押下で自動送りを停止し、再押下で再開する（hover/focus の暫定停止とは独立して、ユーザーが恒常的に止められること）。
    - `prefers-reduced-motion: reduce` 時は初期状態を「停止（自動送りしない）」とし、コントロールは表示してよい（手動切替は可能）。
  - **(b) ドットの不正ARIA修正**: `<button>` への `aria-selected` は無効（`aria-selected` は `option/tab/row/gridcell/treeitem` 等のロール用）。**いずれか一方**へ修正する:
    - 推奨（最小改修）: ドットを**ボタングループのまま** `aria-pressed`（現在スライド=`true`）で表現し、`.fv__dots` の `role="group"`＋`aria-label` を維持する。
    - 代替: `role="tablist"`／各ドット `role="tab"`＋`aria-selected`＋対応パネルへの `aria-controls`／パネル側 `role="tabpanel"` を備えた**正規のタブパターン**に作り替える（採用する場合はキーボード矢印操作も実装すること）。
    - どちらを選ぶ場合も、現在スライドが支援技術に正しく伝わること。各ドットの `aria-label`（「スライドN: …を表示」）は維持する。
  - **(c) h1 動的書換の構造妥当性（推奨案を確定）**: 現状は `h1.innerHTML` をスライドごとに書き換えており、ドキュメントアウトライン／SEO／SR利用者にとって見出しが不安定になる。**推奨案（採用）= 静的h1を維持する**:
    - `h1`（ページの恒常的な大見出し＝スライド1相当のコーポレートコピー）は**固定**し、JSで書き換えない。
    - スライドごとに切り替えるのは **eyebrow（`.fv__eyebrow`）とサブコピー（`.fv__sub`）のみ**とする。
    - 切り替わるテキスト領域（eyebrow＋sub を内包するブロック）に `aria-live="polite"` を付与し、自動更新を SR へ穏当に通知する（多重読み上げ回避のため `h1` は live 対象に含めない）。
    - 代替案（h1書換を残す場合）: 切替テキスト領域を `aria-live="polite"` で包み、かつ (a) の恒常的停止コントロールを必須とする。ただしアウトライン安定性の観点から**静的h1案を優先**する。

**事業紹介（2事業対等）の方針（SPEC §5.3, 受け入れ条件3）:**
- ITインフラ／SES を **同一のカードコンポーネント（同じクラス `.biz-card`・同じ内部構造）** ×2 で並べ、`padding`/`radius`/アイコン寸法を CSS変数（`--card-pad`/`--card-radius`/`--card-icon`）で共通化し面積・視覚的重みを機械的に揃える。説明文の文字量差は `flex:1 1 auto` でリンク位置を揃えて吸収。
- PCは横並び2カラム（`grid-template-columns:1fr 1fr`）、スマホは縦積み。
- **検証用デバッグUI `#bizStack`/`.biz-toggle` は本番DOM・CSSから削除する（D5）。** 併せて `main.js` の scroll-reveal 対象セレクタから `#business .biz-toggle` を除去すること。

**会社案内（company）セクションの方針（新規・D1）:**
- 役割: 理念・会社情報への導線。テキスト＋ビジュアルの2カラム（`min-width:768px` で2カラム、スマホ縦積み）。
- 構造: `h2` 見出し＋リード文＋「会社案内を見る」CTA（`.btn--secondary`）＋ビジュアルプレースホルダ（`.media-photo`、CSSグラデーション＋`aspect-ratio` で枠固定）。
- プレースホルダ方針: 外部依存0。実会社情報・写真は支給後に差し替え前提（リンク先の会社案内ページ本体は別チケット）。

**お知らせ（news）セクションの方針（新規・D1）:**
- 役割: 最新お知らせ3件＋「お知らせ一覧へ」リンク。
- 構造: `h2`＋一覧リンクの見出し行、`<ul>` に `<li>`×3（各 `<time datetime>`＋タイトル）、注記（一覧／個別記事は別途実装）。
- プレースホルダ方針: **ダミーデータ（外部依存0）**。日付は `<time datetime>` で機械可読に。リンク先（一覧・個別記事）は別チケットのためアンカープレースホルダ（`#news`）。実CMS連携時に差し替え前提。

**お問い合わせ（contact）/ フッターの方針:**
- contact: `h2`＋リード＋2CTA（サービス相談／採用問い合わせ）＋注記（フォーム本体は別チケット）。
- footer: ブランド・事業内容・会社情報（お知らせ／会社案内／採用情報）・お問い合わせのリンク群＋コピーライト。`min-width:768px` で4カラム。

**カラートークン（CSS変数 / `:root`、SPEC §5.1。実装済み値を正とする）:**
```css
:root{
  --color-bg:#ffffff; --color-surface:#f6faf7;
  --color-primary:#1f7a4d; --color-primary-dark:#14603b; --color-primary-light:#d8efe2;
  --color-accent:#a85909;  --color-accent-dark:#884706;   /* 白文字 5.12:1（AA） */
  --color-text:#1a2b22; --color-text-muted:#43574c; --color-on-primary:#ffffff;
  --space-section:clamp(48px,8vw,96px); --radius:12px; --maxw:1120px;
  --card-pad:clamp(20px,4vw,32px); --card-radius:var(--radius); --card-icon:56px;
}
```
- 主要テキスト（本文・主要見出し・ボタン文字）はコントラスト比 ≥ 4.5:1（大見出しは最低 ≥ 3:1）。実ブランドカラー支給時はトークン値のみ差し替える。

**CTA / SVG / フォント / JS の方針:**
- CTAは `<a class="btn ...">`。`.btn--primary`（緑）／`.btn--secondary`（アウトライン）／`.btn--accent`（採用の差し色）。タップ領域 `≥44×44px`、`:focus-visible` 可視。計測は `data-cta` 属性のみ（GA4本実装は別チケット）。
- 図形アクセントはインラインSVG（外部画像リクエストを増やさない）。装飾SVGには `aria-hidden="true"`。写真領域はCSSグラデーション＋`aspect-ratio` で枠固定（実写真は差し替えのみで反映）。
- フォントはシステムフォントスタック優先（Webフォント0リクエスト）。
- JS（`main.js`）は依存0・`defer`。`initNavToggle / initCtaTracking / initFvCarousel / initScrollReveal` を初期化。

**scroll-reveal の方針（D3）:**
- 対象要素が画面に入ったら下から24px＋フェードイン（IntersectionObserver・各要素1回のみ）。FVは対象外（カルーセルで既に動く）。事業カード・お知らせ項目はグループ内 stagger（最大360ms）。
- **FOUC回避**: `.reveal-ready` クラスを **JS有効時のみ `<html>` に付与**し、初期非表示（`opacity:0; translateY(24px)`）はこのクラス配下でのみ適用する。JS無効・IO非対応時はクラスが付かず通常表示となり、コンテンツが隠れたままになることを防ぐ。
- **`prefers-reduced-motion: reduce` 尊重**: reduce時は `initScrollReveal` を早期 return し、CSS側でも `.reveal` を `opacity:1; transform:none; transition:none` にフォールバック。
- 対象セレクタから検証用 `#business .biz-toggle` を除去する（D5でUI自体を削除するため）。

**処理フロー（ページ表示時）:**
```
1. ブラウザが site/index.html を取得
2. css/style.css 読込 → カラートークン適用・モバイルファーストで縦積みレイアウト
3. min-width ブレークポイント到達時に FV/事業/採用/会社/フッターが2カラム以上へ
4. インラインSVG・プレースホルダ（CSSグラデーション）を描画
5. main.js（defer）でメニュー開閉・data-ctaフック・FVカルーセル・scroll-revealを初期化
   （reduced-motion時はカルーセル自動送り停止・reveal無効）
```
